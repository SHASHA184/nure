# Лабораторна робота 1: Організація даних та їх структурування

## Мета роботи
Отримання студентами практичних навичок створення та реалізації:
- Простих конструкцій (розгалуження, перемикачі, цикли)
- Масивів, зрізів, карт
- Рядків та структур

## Теоретичні відомості

### 1. Форматування (gofmt)
- Використовувати `gofmt` або `go fmt` для автоматичного форматування коду
- Go має стандартний стиль форматування, який застосовується автоматично
- Приклад: `gofmt` автоматично вирівнює коментарі у структурах

```go
type T struct {
    name    string // name of the object
    value   int    // its value
}
```

### 2. Коментарі
- Однорядкові: `//`
- Багаторядкові: `/* */`
- Коментарі для `godoc` пишуться перед оголошенням без порожніх рядків
- Кожен експортований об'єкт (з великої літери) повинен мати коментар

```go
// Compile parses a regular expression and returns, if successful,
// a Regexp that can be used to match against text.
func Compile(str string) (*Regexp, error) {
```

### 3. Найменування
- **Пакети**: коротке ім'я в нижньому регістрі (одне слово)
- **Експорт**: велика літера = експортується, мала = приватне
- **Геттери**: `Owner()` замість `GetOwner()`
- **Сеттери**: `SetOwner()`
- **Інтерфейси**: ім'я методу + суфікс `-er` (Reader, Writer, Formatter)
- **Стиль**: `MixedCaps` або `mixedCaps`, не підкреслення

### 4. Крапка з комою
- Лексер автоматично додає крапку з комою
- Не потрібно писати вручну
- Відкриваючу дужку `{` не можна переносити на новий рядок

### 5. Керівні структури

#### if
```go
if x > 0 {
    return y
}

// З ініціалізацією
if err := file.Chmod(0664); err != nil {
    log.Print(err)
    return err
}
```

#### for
```go
// Класичний for
for i := 0; i < 10; i++ {
    sum += i
}

// while-подібний
for condition {
}

// Нескінченний цикл
for {
}

// range
for key, value := range oldMap {
    newMap[key] = value
}
```

#### switch
```go
// Без виразу (як if-else-if)
switch {
case '0' <= c && c <= '9':
    return c - '0'
case 'a' <= c && c <= 'f':
    return c - 'a' + 10
}

// З кількома значеннями
switch c {
case ' ', '?', '&', '=', '#', '+', '%':
    return true
}

// Type switch
switch t := t.(type) {
case bool:
    fmt.Printf("boolean %t\n", t)
case int:
    fmt.Printf("integer %d\n", t)
}
```

## Завдання до виконання

Приклади варіантів:

2. Розпізнавання прямокутних трикутників за довжинами сторін
3. Знайти max{min(a,b), min(c,d)}
4. Визначити чи є серед чисел a,b,c таке, що дорівнює d
5. Чи утворюють 4 точки паралелограм
6. Чи лежать 3 точки на одній прямій
7. Подвоїти числа якщо a<b<c
8. Яка точка ближче до a на осі OX
9. Чи можуть числа бути довжинами сторін трикутника
10. Чи поміститься трикутник у колі
... (та інші варіанти)

### Завдання 2: Використання map (25 варіантів)
При написанні програми використовувати мапи.

Приклади варіантів:
1. За номером дня тижня видати кількість уроків
2. По останній цифрі числа визначити останню цифру квадрату
3. За роком і номером місяця визначити кількість днів
4. Для цифри (0-9) вивести її назву
5. За числом (1-12) вивести назву місяця
6. Словесний опис шкільних відміток (1-5)
7. За номером елемента кола (радіус/діаметр/довжина) обчислити площу
8. Елементи прямокутного рівнобедреного трикутника
9. За номером пори року вивести місяці
10. Для числа k (1-99) надрукувати "Мені k років/рік/роки"
... (та інші варіанти)

### Завдання 3: Цикли та обчислення функцій (25 варіантів)
Скласти програму обчислення значення функцій при різних значеннях аргументів, заданих інтервалом зміни і величиною кроку, результат представити у вигляді таблиці.

Приклади функцій:
- Табулювання різних математичних функцій
- Обчислення функцій двох змінних
- Підрахунок кількості разів, коли функція приймає негативне значення
- Визначення координат точок у різних чвертях
- Робота з геометричними фігурами та координатами

### Завдання 4: Обробка одновимірних масивів (25 варіантів)
Створити програму обробки одновимірного масиву.

Операції (кожен варіант містить 4 операції):
- **Видалення**: максимальний/мінімальний елемент, елемент за номером, парні/непарні елементи тощо
- **Додавання**: K елементів на початок/кінець, N елементів починаючи з номера K
- **Перестановка**: перевернути масив, циклічний зсув, поміняти парні/непарні місцями
- **Пошук**: перший парний/негативний, елемент із заданим значенням, середнє арифметичне

Приклад варіанту 1:
- Видалення: максимальний елемент
- Додавання: K елементів в початок масиву
- Перестановка: перевернути масив
- Пошук: перший парний

# Лабораторна робота №2: RSA шифрування з мережевою взаємодією

## Опис

Система клієнт-сервер для обміну зашифрованими повідомленнями з використанням RSA шифрування (2048 біт).

**Сервер**: Python (FastAPI) + `cryptography`
**Клієнт**: Go + `crypto/rsa` (стандартна бібліотека)
**Протокол**: HTTP/JSON через незахищений канал
**Шифрування**: RSA-OAEP з SHA-256

## Архітектура

1. Обидві сторони генерують свої RSA ключі (2048 біт)
2. Клієнт отримує публічний ключ сервера
3. Клієнт відправляє свій публічний ключ серверу
4. Клієнт шифрує повідомлення публічним ключем сервера
5. Сервер розшифровує приватним ключем
6. Сервер шифрує відповідь публічним ключем клієнта
7. Клієнт розшифровує відповідь своїм приватним ключем

## Запуск

### Сервер (Python)

```bash
cd server
pip install -r requirements.txt
python main.py
```

Сервер запуститься на `http://127.0.0.1:8080`

### Клієнт (Go)

В окремому терміналі:

```bash
cd client
go run main.go
```

## Використання

Після запуску клієнта введіть повідомлення і натисніть Enter.
Сервер розшифрує його та відправить зашифровану відповідь назад.

Введіть `exit` для виходу.

## API Endpoints

- `GET /public-key` - отримання публічного ключа сервера
- `POST /exchange-key` - відправка публічного ключа клієнта
- `POST /message` - відправка зашифрованого повідомлення
